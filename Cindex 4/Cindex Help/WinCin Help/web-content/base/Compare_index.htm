Comparing indexes

Cindex can compare one index with another, enabling you not only to see how they differ, but also to change the contents of the index based on the results of the comparison.

Use the Compare Indexes… command (Tools menu) to compare the active index with another that is open (the menu item is dimmed if no second index is open).

Choose the comparison index from the drop-down list, then from the Compare records on drop-down list choose how Cindex will compare records:

  • All fields: Cindex compares headings, subheadings, and the locator.
  • Headings and subheadings only: Cindex compares headings and subheadings, but ignores the contents of the locator field.
  • Locator field if other fields match: Cindex compares locators of records that match on all headings and subheadings; it ignores records that don’t match on headings and subheadings.

When you click Compare, Cindex examines all records except those that are deleted or empty. When comparing records Cindex ignores any differences in labels.

NOTE: For the most accurate comparison both indexes should be sorted using 'simple' alphabetizing rules.

If any records differ, Cindex enlarges the panel to show summary results in three sections:

  • The top section shows how many records are in the active index but not in the other.
  • The middle section shows how many records match (are the same in both indexes according to the comparison rule).
  • The bottom section shows how many records are in the other index but not the active one.

You can modify the active index based on the results of the comparison. Each section presents options, and you can choose any or all of those that are available:

  • To mark the identified records for deletion, check Delete.
  • To form a group from the identified records, check Group.
  • To label (or unlabel) the identified records, choose from the Label drop-down list.
  • To import the identified records from the other index (bottom section), check Import.

Choose all the actions you want, then click Modify to make the changes.

Actions you trigger with Modify affect only the active index. The comparison index is not touched.

Each group created is named with the date and time of creation, prefixed by $– or $= or $+ to identify the type of comparison result that gave rise to it.
